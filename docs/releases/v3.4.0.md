# Heddle v3.4.0

Release date: 2026-06-08

Heddle v3.4.0 adds the first complete Agent Skills foundation. Skills let users
opt into reusable workflow instructions without putting every long procedure in
the agent context up front.

## Highlights

### Agent Skills use progressive disclosure

Heddle now discovers standard `SKILL.md` files from project and user skill
directories, parses only compact catalog metadata at startup, and lets the
agent read full instructions only when an active skill is relevant.

Why this matters:

- agents can see which specialized workflows are available without crowding the
  prompt with every skill body
- full skill instructions stay behind the `read_agent_skill` tool and only load
  after the workspace has activated that skill
- referenced skill resources can be loaded on demand instead of eagerly reading
  entire skill folders

### Skill activation is workspace-scoped

Each workspace now stores skill activation state under
`.heddle/skills/activation.json`. Heddle records only user decisions there; the
skill files remain in their original project or user locations.

Why this matters:

- one workspace can enable a skill without changing another workspace
- users can distinguish not-enabled skills from skills explicitly disabled in
  the current workspace
- project and user skills remain normal inspectable filesystem assets rather
  than copied hidden state

### Skills are manageable from TUI and web

Terminal chat now has shared `/skills`, `/skills enable <name>`, and
`/skills disable <name>` commands. The web control plane also has a Skills
settings page for searching discovered skills, reviewing source/status, and
enabling or disabling them.

Why this matters:

- users have a clear consent surface before skill instructions affect future
  agent turns
- web and terminal surfaces share the same control-plane activation behavior
- catalog issues and missing skill definitions are visible instead of failing
  silently

### Agent Skills are documented as a core domain

This release adds public Agent Skills documentation, programmatic-host notes,
CLI reference updates, and service-level documentation for the core skills
domain.

Why this matters:

- contributors have a clear boundary for discovery, parsing, activation state,
  runtime catalog injection, and read-tool behavior
- host integrations can reuse the same state and progressive-disclosure model
- skills remain instructions, not permissions; existing Heddle approval and
  runtime safety policy remains authoritative

## Upgrade Notes

- No migration is required from v3.3.0.
- `heddle` and `heddle chat` continue to open the API-backed TUI.
- Add project skills under `.agents/skills/<name>/SKILL.md`.
- Add user skills under `~/.agents/skills/<name>/SKILL.md`.
- Use `/skills` in terminal chat or the web Settings → Skills page to activate
  skills for a workspace.
- Packaged Heddle built-in skills are not included yet; this release provides
  the foundation for them.
