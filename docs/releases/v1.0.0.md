# Heddle v1.0.0

Release date: 2026-05-01

Heddle v1.0.0 marks the first 1.0 release of the runtime and CLI.

This release packages a large body of work that makes Heddle more usable as a real coding/workspace agent platform: a programmatic conversation engine, a clearer toolkit architecture, stronger chat-turn/runtime boundaries, better control-plane session UX, and a first local evaluation harness for repeatable agent tasks.

## Highlights

### Programmatic conversation engine alpha

Heddle now exposes an alpha `createConversationEngine` surface for hosts that want to run persisted conversations outside the terminal UI.

This adds a clearer host-facing API for session storage, callbacks, and turn orchestration, alongside docs and examples that explain how to embed the runtime.

Why this matters:

- Heddle is more clearly usable as an evolving coding/workspace agent runtime, not only a single terminal app
- external hosts have a better path to integrate persisted agent conversations
- the shared runtime boundary is easier to reason about and extend

### Toolkits and command/runtime boundary cleanup

The runtime’s tools are now organized through toolkit-owned modules, with clearer shell-process policy boundaries and more modular slash-command handling.

This release also continues the larger chat-turn cleanup work by extracting preflight, persistence, memory-maintenance, host-bridge, and runtime responsibilities into more focused units.

Why this matters:

- runtime internals are easier to maintain and evolve safely
- tools and command behavior are easier to test in isolation
- future host surfaces can share more of the same core behavior

### Better control-plane and session workflows

The browser control plane received substantial session-screen and review-surface improvements, including stronger live-session subscriptions, cleaner session mutations, better model/session handling, and a more modular screen structure.

Why this matters:

- operators get a clearer multi-session workflow in the browser UI
- session switching and review behavior are more consistent with the shared runtime
- the control plane is in better shape for continued product work

### First local eval harness

Heddle now includes a first pass at a local coding-agent evaluation workflow with `heddle eval agent` and `heddle eval clean`, plus fixture-backed cases and inspectable reports.

Why this matters:

- agent behavior can be judged against repeatable tasks instead of only ad hoc sessions
- runtime and prompt changes have a more concrete place to prove their value
- local dogfooding and regression checking are easier to inspect and clean up

### Documentation and reliability improvements

This release includes stronger programmatic-use documentation, example conformance hardening, regex-heavy prompt rendering fixes, and additional coverage across conversation activity, approvals, slash commands, chat-turn runtime behavior, and control-plane screens.

Why this matters:

- public docs better match the implementation surface
- more edge cases are covered by tests
- day-to-day agent usage is less likely to fail on formatting or orchestration issues
