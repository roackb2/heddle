# Heddle v1.3.0

Release date: 2026-05-15

Heddle v1.3.0 continues the conversation-engine boundary cleanup and makes
one-shot ask mode behave like the rest of Heddle's persisted conversation
surfaces instead of following a separate runtime path.

The main theme is convergence: fewer host-specific branches, clearer
controller/service boundaries, and a more consistent session model across CLI
ask, terminal chat, and the control plane.

## Highlights

### Ask mode now runs through persisted one-off sessions

`heddle ask` still behaves like a one-shot terminal command, but it now records
the run as a one-off saved session under `.heddle/` instead of composing its
own raw execution path.

Why this matters:

- ask mode now shares turn persistence, traces, and memory maintenance with the
  normal conversation engine
- daemon-backed ask and local ask follow the same session/turn flow
- future ask features can be added at the engine boundary instead of being
  reimplemented in host-specific branches

### Control-plane chat boundaries are cleaner

This release continues the control-plane cleanup by moving chat-session
behavior behind dedicated controllers and core conversation services rather
than mixing transport-facing logic with session ownership.

Why this matters:

- browser and daemon flows now have a clearer boundary for session creation,
  turn submission, and review data shaping
- future contributors have a more legible place to extend control-plane chat
  behavior
- boundary cleanup reduces the chance of new host-only branches reappearing in
  session code

### Session services and verification coverage are stronger

The underlying session service and verification surfaces were extended so more
session behavior lives behind one reusable core module, with targeted tests and
local verification scripts covering the expected TUI/control-plane flows.

Why this matters:

- session semantics are less likely to drift between TUI, ask mode, and the
  browser control plane
- repository-backed session behavior is easier to refactor without host-layer
  file access leaking back in
- e2e coverage now protects more of the browser workstation path

## Other Changes

- Added `yarn lint` as the public alias for the existing eslint command.
- Renamed the TUI verification script to `verify:tui-behavior` and kept the
  older script names as aliases.
- Updated public docs so CLI ask behavior and programmatic entrypoint guidance
  match the current architecture.
