# Heddle v1.8.0

Release date: 2026-05-19

Heddle v1.8.0 introduces the first public foundation slice for the new
browser control plane (`src/web-v2`). The focus is on a clean frontend
architecture so future control-plane work can proceed from a dedicated,
stable shell instead of extending the legacy surface.

## Highlights

### `src/web-v2` now has a working v2 shell

The new control-plane path now includes a bounded app frame, route setup, session
sidebar, and a lightweight conversation workspace so the UI can support
workflow-driven growth one surface at a time.

Why this matters:

- v2 work no longer depends on legacy `src/web` coupling patterns
- navigation and shell primitives are separated from workflow panels
- upcoming browser features can be built with small incremental view-level surfaces

### Browser integration now covers v2 layout

Browser integration coverage was extended with a v2 control-plane runner and
project-level Playwright wiring so `yarn test:browser-integration` validates both
legacy and v2 UI flows.

Why this matters:

- control-plane browser checks are less biased toward one UI surface
- v2 regressions are detectable before merge
- the runner path is reusable for additional UI projects

### `client:v2` scripts and v2 primitives

The package now includes dedicated scripts for building and running the v2
frontend and adds v2-owned shadcn/React primitives and i18n boundaries for
localized shell strings.

Why this matters:

- v2 has a self-contained build/development entrypoint
- shared contracts stay inside the v2 boundary
- localization and UI behavior can evolve before the broader v2 work is complete
