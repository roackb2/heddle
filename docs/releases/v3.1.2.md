# Heddle v3.1.2

Release date: 2026-06-05

Heddle v3.1.2 is a focused usability and recovery release for the shared
control-plane experience. It makes the terminal review flow easier to operate,
keeps browser and terminal activity presentation aligned, and prevents stale
session leases from blocking users for too long after a process dies mid-run.

## Highlights

### TUI diff review is easier to use

The terminal UI now has a dedicated recent-edit diff review flow instead of
leaving large diff blocks mixed into the main prompt area.

Why this matters:

- recent file edits can be reviewed from the terminal without switching to the
  browser
- diff rendering is kept in TUI presentation components instead of duplicating
  core behavior
- long review content no longer makes it as hard to tell when the run itself has
  finished

### Tool activity appears in the conversation timeline

Tool-call activity now has a shared presentation shape that both web-v2 and the
TUI can render inside the conversation flow.

Why this matters:

- approvals and tool activity sit beside the assistant response they belong to
- web and terminal surfaces share the same presentation semantics
- future collapse/expand behavior has a cleaner data foundation

### Prompt typing is more responsive

The TUI prompt composer now keeps prompt input local and lighter-weight while
the rest of the screen is busy rendering run output.

Why this matters:

- typed characters are less likely to lag during active runs
- long prompts wrap more predictably
- the input area is split from broader app rendering concerns

### Stale session leases recover quickly

Active control-plane runs now refresh their session lease while they are alive.
If the owning process dies mid-run, the lease becomes reclaimable after roughly
20 seconds instead of blocking the session for much longer.

Why this matters:

- killing or losing a server/TUI process no longer leaves a session unusable for
  an extended period
- live runs still protect sessions from unsafe concurrent mutation
- recovery behavior is owned by the session lease policy and server run
  lifecycle, not by individual clients

## Upgrade Notes

- No command migration is required from v3.1.1.
- `heddle` and `heddle chat` continue to open the API-backed TUI.
- Browser and terminal clients continue to use the shared control-plane path.
