# Heddle v0.0.38

Release date: 2026-05-01

Heddle v0.0.38 is a release focused on making real agent work easier to evaluate, easier to test, and less likely to stall on unnecessary runtime pressure.

This release also updates `main` to the next npm version after the already-published `v0.0.37` release.

## Highlights

### Agent evaluation harness

Heddle now includes a first pass at a local coding-agent evaluation workflow.

This adds:

- `heddle eval agent` for running JSON-defined coding eval cases
- `heddle eval clean` for pruning generated eval result directories
- fixture-backed smoke cases for fixing failing tests, targeted refactors, and verification discipline
- dogfood roadmap fixtures for larger Heddle-specific milestones
- markdown and JSON eval reports with trace, diff, and check summaries

Why this matters:

- release candidates can be judged against repeatable agent tasks instead of only ad hoc manual sessions
- future runtime and prompt changes have a clearer place to prove they still improve real coding behavior
- eval artifacts are easier to inspect and clean up after local dogfood runs

### Test suite taxonomy

The test tree is now organized by layer and surface.

The package scripts split the baseline into:

- `yarn test:unit`
- `yarn test:integration`
- `yarn test:e2e`

The default `yarn test` command runs the unit and integration suites, while Playwright remains available through the explicit e2e scripts.

Why this matters:

- the normal release baseline is easier to understand
- slower browser coverage is separated from the core verification path
- future CI and local debugging can target the relevant layer more directly

### Model compatibility and selector polish

OpenAI OAuth model selection is now centralized around model compatibility policy.

The terminal and control-plane model selectors have clearer behavior for models that are unavailable or inappropriate for the active credential mode, with a more polished browser selector and compacting status animation.

Why this matters:

- OAuth account-sign-in users get fewer dead-end model choices
- model picker behavior is easier to keep consistent across UI surfaces
- session compaction state is more visible during model/context transitions

### Runtime prompt pressure cleanup

Several older prompt/runtime pressure mechanisms were removed or softened.

This release removes the `report_state` tool, trims mutation-tracking leftovers, and reduces progress/post-mutation reminders that were making the agent over-report or stop around intermediate implementation details.

Why this matters:

- agents can focus more on completing the user’s task
- the runtime has fewer artificial checkpoints that compete with actual verification
- the system prompt is shorter and more direct

### Shared session behavior fixes

The shared chat/session path has additional fixes around control-plane continuation and session switching.

This includes preserving model choice across sessions, updating session-switch content correctly, and aligning control-plane continue behavior with the shared runtime.

## Operator notes

- Package version is now `0.0.38`.
- Latest already-published npm version before this release was `0.0.37`.
- No breaking CLI migration is required for normal chat or ask usage.
- `heddle eval` is an early operator/developer workflow and should be treated as a dogfood aid, not a complete public benchmark suite.
- The normal release baseline remains `yarn build`, `yarn test`, and `npm pack --dry-run --cache /tmp/heddle-npm-cache`.
