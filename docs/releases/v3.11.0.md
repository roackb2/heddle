# Heddle v3.11.0

Heddle v3.11.0 introduces custom agents: named, turn-scoped roles that combine
prompt appendices, tool visibility, approval behavior, and runtime defaults.
The built-in Ask, Code, and Review modes now sit on the same foundation that
users can extend with project-specific or user-level agents.

## What's Changed

- Added the custom agents core domain with built-in Ask, Code, and Review
  definitions plus support for `.agents/agents/<id>/AGENT.md` and
  `~/.agents/agents/<id>/AGENT.md` files.
- Added runtime tool profiles and approval profiles so agents can be scoped to
  inspection-only, coding, read-only, interactive, or trusted-auto behavior.
- Made chat turns persist the selected agent, keeping agent choice turn-scoped
  instead of binding one profile to an entire saved session.
- Added CLI support for selecting agents and modes from `heddle ask`.
- Added browser control-plane settings for creating, viewing, editing, and
  deleting project agents.
- Added composer controls for switching between Ask, Code, Review, and custom
  agents before sending the next prompt.
- Added focused unit coverage and a `smoke:custom-agents` verification script
  for the custom agent path.
- Documented custom agents in the README, zh-TW README, docs hub, and dedicated
  guide.

## Workflow Notes

Custom agent definitions live in standard markdown files with YAML frontmatter:

```text
.agents/agents/<agent-id>/AGENT.md
~/.agents/agents/<agent-id>/AGENT.md
```

Use built-in modes for the common path:

```bash
heddle ask --mode ask "Explain the current workspace"
heddle ask --mode review "Review the current diff"
```

Use a project or user agent by id when a role needs dedicated instructions,
tools, approval behavior, or runtime defaults:

```bash
heddle ask --agent repo-reviewer "Review the current workspace changes"
```

In the browser control plane, project agents can be managed from
`Settings -> Agents`, and the chat composer plus menu can choose the agent for
the next turn. Agent selection is intentionally turn-scoped, so a saved session
can move from Ask to Code to Review without starting a new conversation.

Custom agents do not bypass Heddle policy. Tool visibility and approval behavior
are still enforced by the runtime-owned tool and approval profiles.
