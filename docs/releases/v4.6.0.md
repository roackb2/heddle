# Heddle v4.6.0

Heddle v4.6.0 lets hosted products distinguish exhausted model quota or billing
from transient rate limiting. Hosts can now give users the correct recovery
action without parsing provider messages or exposing provider diagnostics.

This release is additive. Existing hosts can ignore the new `quota` failure
code; hosts that project typed model failures can handle it separately from
retryable `rate_limit` failures.

## Quota-aware model failures

- **Quota has a stable public code.** `ModelRunFailureCode` now includes
  `quota`, and the code propagates through runtime results, conversation turn
  summaries, persisted artifacts, and hosted contracts.
- **Structured provider codes win over missing status.** A structured
  `insufficient_quota` code is recognized from direct or nested SDK errors even
  when the provider does not attach a numeric HTTP status.
- **Transient rate limiting remains retryable.** Ordinary HTTP 429 failures
  continue to resolve as `rate_limit`; quota and billing exhaustion are
  non-retryable.
- **Provider text remains private.** Message content alone never triggers quota
  classification, and raw provider bodies or diagnostics are not projected into
  public results.

## Upgrade notes

- Install `@roackb2/heddle@4.6.0` to consume the new `quota` failure code.
- Hosts with exhaustive `ModelRunFailureCode` handling should add a `quota`
  branch with billing, quota, or credential-replacement guidance.
- `@roackb2/heddle-remote@4.6.0` is released in lockstep. Its runtime protocol
  is unchanged in this release.
