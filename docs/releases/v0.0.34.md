# Heddle v0.0.34

Heddle v0.0.34 is a follow-up patch release that carries the latest completion-oriented system-prompt fix into the shipped runtime after `v0.0.33` was already cut.

## Highlights

- Strengthened the top-level Heddle system prompt so the agent is explicitly told to carry clear, bounded tasks through to completion
- Added direct guidance that Heddle should only stop to ask the user when something is genuinely unclear
- Preserved the broader v0.0.33 behavior improvements around OpenAI OAuth `view_image`, direct delete/move file tools, and softer blocker/report-state guidance

## Why it matters

This patch is specifically about follow-through.

The project has been tightening Heddle’s behavior so it acts more like a real coding/workspace operator and less like a tool wrapper that pauses after every intermediate finding. The prompt adjustment in this release reinforces that direction at the highest level:

- when the task is clear, keep going
- do not wait for repeated nudges
- only ask again when something is truly unclear

## Operator notes

- This release is intentionally small and targeted.
- It should make Heddle less likely to stall after intermediate inspection or partial progress in clear bounded tasks.
- Existing verification, review, and blocker behavior from `v0.0.33` remains in place.

## Upgrade notes

- No breaking CLI migration is required for this release.
- Existing API-key and OpenAI OAuth workflows continue unchanged.
- This release mainly affects agent behavior and task-carrying semantics rather than command syntax.

## Remaining limits

- Memory retrieval/discovery smoothing is still an active follow-up area.
- OpenAI hosted `web_search` and drift embeddings still require Platform API-key mode.
