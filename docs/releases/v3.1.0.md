# Heddle v3.1.0

Release date: 2026-06-04

Heddle v3.1.0 completes the first major cleanup after the v3 launch: the
legacy terminal UI and legacy browser UI are removed, and the supported command
and browser surfaces now run through the cli-v2/web-v2 control-plane path.

## Highlights

### Legacy CLI and browser UI are retired

The old `src/cli` terminal implementation and `src/web` browser control plane
have been removed from the active package. `heddle`, `heddle chat`, command
edges, and browser control-plane work now route through `src/cli-v2`,
`src/server`, and `src/web-v2`.

Why this matters:

- terminal chat no longer carries the duplicated v1 runtime logic
- command-line behavior has one maintained entrypoint under cli-v2
- browser development no longer has a parallel web-v1 surface to keep in sync
- future contributors have fewer stale paths to accidentally extend

### Command edges use clearer service boundaries

Auth, init, ask, heartbeat, memory, daemon, and eval command behavior now lives
under cli-v2 command-edge services. The commands own terminal parsing,
presentation, and orchestration, while core services and the control-plane API
own runtime behavior and persisted semantics.

Why this matters:

- one-shot ask uses the same control-plane session API path as the TUI
- heartbeat commands run through the server-backed scheduler model
- memory commands expose clearer help and credential behavior
- daemon commands attach to an existing control-plane server when available

### TUI review ergonomics are better

The terminal UI now includes a recent-edit diff panel so users can inspect the
files the agent just touched without leaving the TUI. Long diffs are kept
bounded for terminal readability.

Why this matters:

- terminal users can review recent agent edits inline
- the browser can still show the broader workspace diff view
- the TUI and browser now cover complementary review workflows

### Release and verification guardrails are tighter

Tests and docs were updated around the new active surfaces. The repo now has
guardrails to keep retired cli/web-v1 paths from being reintroduced, and the
TUI behavior verification script is retained against the v2 path.

## Upgrade Notes

- `heddle` and `heddle chat` continue to open the API-backed TUI.
- The legacy terminal UI escape hatch is no longer available.
- The active browser UI is `src/web-v2`; `src/web` has been removed.
- `client:dev` and `client:build` target web-v2.
- `daemon:dev` serves built web-v2 assets from `dist/src/web-v2`.
