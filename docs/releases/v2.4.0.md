# Heddle v2.4.0

Release date: 2026-05-31

Heddle v2.4.0 tightens the API-backed terminal UI and browser control plane
around day-to-day coding flow. This release adds file mentions in `chat-v2`,
shared prompt editing behavior, prompt history navigation, terminal OAuth
parity, and a synced queued-message flow for active agent runs.

## Highlights

### Queued follow-up prompts are synced across devices

When a session is already running, users can now send follow-up prompts instead
of being blocked by a "run already in progress" error. Queued prompts are stored
with the core chat session, exposed through the control plane, and drained in
order after the active turn finishes.

Why this matters:

- users can keep working while an agent turn is active
- queued prompts are visible and editable from web-v2 and `chat-v2`
- the queue is session state, so browser and terminal surfaces stay consistent

### cli-v2 supports file mentions

The API-backed terminal UI now supports file mention suggestions in prompt
input. The mention picker consumes the control-plane file-search API and uses
shared client-side mention formatting instead of importing core services into
the TUI.

Why this matters:

- terminal users can reference workspace files without manually typing paths
- file mention behavior stays aligned with web-v2
- cli-v2 continues to consume API contracts rather than local core policy

### Prompt editing is more capable and shared

Prompt input now supports richer line editing, including word deletion,
cursor movement, undo/redo, and previous-prompt navigation. Browser and
terminal surfaces share prompt input helpers where the behavior is interface
agnostic.

Why this matters:

- repeated coding prompts are faster to revise
- cli-v2 moves closer to v1 terminal editing parity
- web-v2 and cli-v2 avoid drifting on prompt history behavior

### Terminal OAuth can run through cli-v2 slash commands

The `chat-v2` slash command execution path can now use the shared credential
command service for OpenAI OAuth login/status behavior, instead of telling users
to leave the TUI for a separate terminal command.

Why this matters:

- terminal auth recovery works in the active chat surface
- credential command behavior has a clearer core owner
- cli-v2 moves another v1-only workflow into the API-backed path
