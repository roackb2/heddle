# Heddle v3.8.0

Heddle v3.8.0 makes saved session management more practical across the browser
control plane and terminal chat. Users can now rename sessions inline, pin
important work above normal recent sessions, and archive sessions that should no
longer appear in the active session list.

## What's Changed

- Added browser session-list actions for inline rename, pin/unpin, and archive.
- Added an undo toast for newly archived sessions so accidental archive actions
  can be reversed immediately.
- Added persisted session pin and archive state to the shared chat session
  record so browser and terminal surfaces use the same ordering and visibility
  semantics.
- Added terminal slash commands for pinning and unpinning the current session.
- Refactored the browser session-list implementation into focused hooks and
  components for rename, archive, grouped rendering, and item presentation.
- Added user-facing documentation for session rename, pin, unpin, archive,
  archive undo, and archived-session visibility behavior.

## Workflow Notes

Pinned sessions appear before normal recent sessions. Archived sessions are
hidden from the normal session list. In the browser control plane, right-click a
session row to manage it; in terminal chat, use `/session pin` and
`/session unpin` for the current session.

Existing saved sessions continue to work. Sessions without explicit pin or
archive state are treated as unpinned and active.
