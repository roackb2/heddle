# Heddle v1.10.0

Release date: 2026-05-23

Heddle v1.10.0 continues the web-v2 control-plane line with a more practical
review workspace. The release adds a resizable/collapsible inspector, a compact
Monaco-backed diff preview, cleaner typography, and a reorganized web-v2 style
system. The README now shows the v2 control-plane direction while clearly
marking it as work in progress and not yet released.

## Highlights

### Web-v2 side panels are resizable and collapsible

The web-v2 shell now lets operators resize the left session rail and the right
context inspector. The right inspector can also collapse, matching the left
panel behavior.

Why this matters:

- operators can make room for the conversation or the review inspector as needed
- the header and composer stay fixed while only the transcript scrolls
- scrollbars are visually hidden in the main scroll regions without disabling
  native scrolling behavior

### Diff preview is available in the right inspector

The right inspector now shows current workspace changes through the existing
control-plane diff APIs. Each changed file can expand into a compact
Monaco-backed inline preview with one line-number gutter, red deletion rows,
green addition rows, and a raw patch fallback when structured hunks are not
available.

Why this matters:

- web-v2 can inspect file changes without borrowing the legacy web-v1 review UI
- diff state comes from the existing `workspaceChanges` and `workspaceFileDiff`
  API shapes
- session tool events invalidate workspace diff queries so the inspector can
  refresh after tool execution

### Web-v2 typography and styles are now organized

The v2 client now uses a self-hosted Inter variable font and a small
purpose-based typography scale. Shared type roles such as panel titles, nav
labels, body text, captions, and code text are defined once and reused across
the visible v2 surfaces.

The large `tailwind.css` entrypoint is now split into focused CSS files under
`src/web-v2/styles/` for theme tokens, typography, shell primitives,
navigation, conversation, approvals, diff preview, and composer styling.

Why this matters:

- visible text has a clearer size and weight system
- future v2 UI work has a maintainable place for reusable style primitives
- Tailwind v4 tokens remain centralized while surface-specific CSS is easier to
  review

### README shows the web-v2 direction

The root README now uses the new web-v2 control-plane screenshot for the
control-plane overview and explicitly labels that interface as work in progress
and not yet released.
