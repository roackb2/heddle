# Heddle v3.12.0

Heddle v3.12.0 adds desktop notifications for control-plane work that needs
operator attention. Browser and terminal users can now be notified when a
session needs approval or when a run finishes, which makes long-running agent
work easier to monitor away from the active tab or terminal.

## What's Changed

- Added shared notification intent projection for approval requests, session
  completion, and heartbeat task completion or failure.
- Added browser notification delivery with permission handling, toast fallback,
  and background tab attention marking.
- Added terminal notification delivery for cli-v2, using Warp's native terminal
  notification escape sequence when running inside Warp and native macOS
  notifications as a fallback outside Warp.
- Added workspace-level session event subscriptions so the browser can notify
  for sessions in the open workspace instead of only the selected session.
- Added cli-v2 polling fallback notification delivery so terminal completion
  notifications still fire when polling observes the active run finish.
- Added Notification settings copy in the web control plane and documented the
  terminal notification behavior in CLI and control-plane references.
- Removed the redundant `node-notifier` / `terminal-notifier` dependency path
  after validating Warp and browser-native notification behavior.

## Workflow Notes

Browser notifications require browser permission and are delivered by the
browser profile that has the Heddle control plane open. They can still be
affected by the operating system notification style, Focus mode, and browser
notification settings.

In Warp, cli-v2 uses Warp's terminal-native notification channel so completion
and approval notifications appear from Warp itself. Non-Warp macOS terminals use
the native macOS notification bridge as a fallback. Heddle also sends a terminal
attention signal, but audible behavior still depends on the terminal and system
settings.

This release does not add closed-client push delivery. Notifications are active
while a browser control-plane workspace or cli-v2 session is running.
