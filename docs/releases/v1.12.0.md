# Heddle v1.12.0

Release date: 2026-05-25

Heddle v1.12.0 is a feature release for the web-v2 control-plane line and the
heartbeat task runtime. It adds a real task workbench, explicit task
continuation controls, live task run updates, memory settings, browser image
uploads, and a more capable session composer with drift and file-mention
controls.

## Highlights

### Web-v2 now has a heartbeat task workbench

The web-v2 control plane can create, edit, enable, disable, delete, run, and
resume heartbeat tasks. Operators can inspect selected task detail, follow live
run state, and open saved run records from the browser.

Why this matters:

- recurring work is no longer only visible as passive task state
- blocked or paused tasks require an explicit resume action
- task views, run views, and browser actions use the same heartbeat task service
  as the CLI and programmatic scheduler APIs

### Task continuation policy is explicit

Heartbeat tasks can now record whether continuation is operator-controlled or
agent-selected. The browser task dialog exposes that choice alongside schedule,
model, and step-budget controls.

Why this matters:

- autonomous continuation is a deliberate task setting instead of hidden policy
- operators can keep blocked tasks under manual control
- task state is easier to explain from saved records and live events

### Web-v2 session composer is more practical

The session composer now includes drift controls, debounced `@file` mention
suggestions, execution settings, better autosizing and scrolling behavior, and
image attachment controls. File suggestions respect ignore rules, and uploaded
browser images are saved under the active workspace state root and appended to
the prompt as local paths for the existing `view_image` workflow.

Why this matters:

- screenshots and reference images can enter browser sessions without a separate
  image-analysis path
- file mentions help steer inspection without blindly pasting file contents
- composer controls are closer to the terminal workflow while staying
  browser-native

### Memory and settings are visible in web-v2

The web-v2 settings area now includes general settings, workspace settings, and
memory status. Memory status shows catalog health, note counts, pending memory
candidates, and the latest maintenance run when available.

Why this matters:

- memory health can be checked from the browser control plane
- settings are separated from the main agent workbench
- workspace and memory surfaces are easier to find during operator review

### Session and task subscriptions were hardened

The control-plane server now has shared runtime subscription stream
infrastructure for tRPC subscriptions, and web-v2 session/task hooks are grouped
by purpose. Session auto-title behavior is unified, session-list events refresh
browser sidebars, and live task events keep run selection in sync.

Why this matters:

- live browser state is less dependent on ad hoc controller-local queues
- session and task updates share clearer transport behavior
- web-v2 code is easier to extend without growing one large app-state hook
