# Heddle v1.0.2

Release date: 2026-05-09

Heddle v1.0.2 is a patch release focused on making repository search and approval prompts more reliable during everyday coding sessions.

The main theme is fewer dead ends: Heddle can keep searching even when `rg` is unavailable, its fallback search path respects normal project ignore behavior more closely, and approval prompts now show the action being remembered or approved more clearly.

## Highlights

### Search works in more environments

`search_files` still prefers `rg` for fast repository search, but now falls back to `grep` when ripgrep is missing or unavailable in the active environment.

Why this matters:

- lightweight machines and constrained agent environments are less likely to lose basic search
- ignored, generated, and protected folders are handled more consistently across backends
- explicitly targeting ignored folders still works when that is the intended search scope

### Approval prompts are easier to trust

Terminal approval flows now label search and shell requests more clearly, including the search query or command under review.

Why this matters:

- users can tell what a remembered approval will actually cover
- repeated safe actions can be approved with less ambiguity
- approval history and regression coverage better match the visible terminal behavior

### Agent docs are more portable

Project agent instructions were trimmed so public repo guidance no longer assumes private companion notes are present.

Why this matters:

- new contributors can follow the repo-local docs without maintainer-specific context
- optional private workspace notes remain supported, but are documented as optional
- public instructions stay aligned with the open-source project boundary

## Other Changes

- Normalized the official website URL in package metadata and docs.
- Replaced the browser favicon with a portable SVG asset.
- Added regression coverage for search fallback behavior, approval formatting, remembered approval labels, and chat formatting.
