# Heddle v0.0.35

Heddle v0.0.35 fixes two practical issues found during real CLI use: long chat sessions can now compact reliably again, and OpenAI OAuth login on Windows avoids shell parsing that could break the authorization URL.

## Highlights

- Fixed chat history compaction for OpenAI OAuth sessions by choosing a summarizer model that works with the Codex backend transport
- Added a centralized model policy for Heddle-chosen internal models, including validation for provider/auth combinations
- Changed chat compaction from a raw message-count cutoff to a rough character-budget selection, so compaction is based on actual prompt size pressure
- Fixed Windows OpenAI OAuth browser launch by avoiding `cmd.exe` URL parsing and passing the authorization URL through a PowerShell encoded command
- Kept the user goal out of the system prompt so goal text is sent as normal user input after the stable system prompt
- Added regression coverage for compaction sizing/model selection and Windows OAuth URL launching

## Why it matters

This release is mostly about keeping real Heddle sessions unblocked.

The compaction fix prevents long chat sessions from getting stuck after Heddle tries to summarize earlier history. This is especially important for OpenAI account sign-in, where the OAuth/Codex backend has different request constraints than the Platform API-key path.

The Windows OAuth fix addresses the first external user report for Heddle. On Windows, the previous browser launcher could let shell parsing split OAuth query parameters such as `&`, causing OpenAI to show `missing_required_parameter` before Heddle could receive the callback.

## Operator notes

- OpenAI OAuth users should see fewer context-window failures in long CLI chat sessions.
- Windows users should be able to run `heddle auth login openai` without the authorization URL being mangled by the shell opener.
- Internal Heddle model choices now go through `src/core/llm/model-policy.ts` instead of scattered conditional logic.
- The system prompt remains more cache-friendly because the per-task goal is no longer embedded in it.

## Upgrade notes

- No breaking CLI migration is required for this release.
- Existing API-key and OpenAI OAuth credentials continue to work unchanged.
- If you are already running `0.0.34`, this is a patch upgrade focused on compaction and Windows OAuth reliability.

## Remaining limits

- OpenAI hosted `web_search` still requires Platform API-key mode.
- Drift embeddings remain API-key-only under the current OpenAI path.
- The Windows OAuth launch fix is covered by command-shape tests, but the end-to-end browser callback still benefits from real Windows user verification after release.
