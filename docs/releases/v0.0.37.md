# Heddle v0.0.37

Heddle v0.0.37 makes the shared conversation runtime much clearer, improves chat workflow stability, and fixes a Windows OAuth ask-flow crash while tightening memory judgment guidance.

## Highlights

- Extracted a shared ordinary chat-turn runtime so TUI and control-plane/server no longer maintain separate ordinary-turn execution flows
- Thinned the TUI chat runtime into more focused adapters for lifecycle, result handling, compaction status, direct shell, approval flow, and status-summary derivation
- Improved prompt input and approval details in chat, including better path-aware approval messaging and more stable prompt input behavior
- Fixed OpenAI OAuth ask-flow credential resolution on Windows so workspace `.heddle` state directories are not mistaken for the provider auth store file
- Added memory-judgment prompt guidance so memory decisions rely more on explicit reasoning and reusable examples
- Recompact chat history automatically when switching to a smaller built-in model context window, with a fix to avoid model ping-pong on session switch

## Why it matters

This release continues turning Heddle from a TUI-first implementation into a more reusable conversation engine.

The biggest architectural change is the shared ordinary-turn runtime extraction. Core chat/session semantics now live more clearly in shared code, which reduces duplication between the terminal chat experience and the browser/control-plane path. That makes future multi-client and programmatic host work more practical.

On the user-facing side, chat sessions should feel steadier: approval details are clearer, prompt input is more reliable, and smaller-model switches are less likely to leave history oversized for the new context window.

The Windows OAuth fix also removes a real crash path in `heddle ask` for OpenAI sign-in users.

## Operator notes

- TUI and control-plane ordinary chat turns now rely more heavily on shared core chat runtime modules under `src/core/chat/`
- Session status/footer/active-turn summary logic is now centralized in `useChatStatusSummary`
- OpenAI OAuth ask/runtime flows now resolve credential store paths correctly instead of treating workspace state directories as auth files
- Switching to a smaller built-in model context window triggers recompaction of chat history for the active session

## Upgrade notes

- No breaking CLI migration is required for this release.
- Existing sessions and credentials should continue to work.
- If you are developing against Heddle internals, expect cleaner separation between shared chat runtime code and TUI adapters.

## Remaining limits

- OpenAI hosted `web_search` still has provider/runtime caveats that should continue to be treated carefully during real-use verification.
- Some browser/control-plane parity work remains a follow-on even though the shared runtime boundary is materially improved in this release.
