# Heddle v3.13.0

Heddle v3.13.0 makes Browser Automation more useful for real web tasks by
adding a native Chrome CDP backend and a per-message browser intent in the web
composer. Users can prepare a Heddle-owned Chrome profile for logged-in sites,
then ask the agent to browse or inspect the relevant page from the normal chat
flow instead of starting from a diagnostic launcher.

## What's Changed

- Added a native Chrome CDP backend that can attach to locally installed Chrome
  launched with a Heddle-owned profile and remote debugging endpoint.
- Added the native Chrome profile launcher, settings controls, slash commands,
  and connection checks for preparing logged-in browser sessions.
- Added a one-shot composer "Use browser" intent so a user can nudge the next
  message toward browser inspection or website interaction without turning the
  setting page into the task launcher.
- Added core-owned browser intent context so UI surfaces send a structured flag
  instead of duplicating model instructions.
- Preserved browser intent through queued follow-up prompts.
- Let native Chrome auto-launch from the actual `browser_open` URL when the CDP
  endpoint is not already reachable.
- Documented browser service boundaries across settings, native Chrome launch,
  CDP attach, browser intent, and the browser research toolkit.
- Replaced placeholder validation defaults with neutral Wikipedia examples.

## Workflow Notes

Browser Automation remains opt-in. Enable it for the workspace before expecting
default agent turns to see browser tools.

For logged-in sites, prepare a profile first from Settings -> Browser
Automation or `/browser launch-native [url]`, complete login in the visible
Chrome window, and keep that Chrome window open while the agent attaches
through CDP. The native Chrome backend is designed for sites that reject
Playwright-launched browsers, but it still follows Heddle's browser policy and
approval boundaries.

For ordinary tasks, ask in chat. Use the composer plus menu -> Use browser only
when the next message should prefer browser inspection or website interaction.
The agent should still choose the relevant page from the user request or task
context; browser code must stay site-agnostic.

Native Chrome CDP click support remains intentionally limited until it can
preserve the same navigation-policy guarantees as the Playwright backend.
