# Heddle v5.1.0

Heddle v5.1 makes production conversation storage easier to implement and
verify. Hosts can now inject durable archive storage alongside their session
repository, reuse Heddle's persistence codecs and pagination rules, and run the
same repository conformance scenarios that protect the built-in adapter.

## Adapter-authoring kit

- **Shared persistence behavior reduces host duplication.** Public codecs own
  full-record validation and catalog projection, while shared pagination
  helpers own page limits, cursor encoding, and canonical ordering.
- **A reusable conformance suite verifies real adapters.** Eight runner-neutral
  scenarios cover CRUD and revisions, two-instance conflicts, stable tied-row
  pagination, filters, tenant isolation, fresh-instance recovery, corruption,
  and invalid page limits.
- **The built-in file repository uses the same primitives.** Heddle's default
  adapter and custom database adapters now share the behavior that defines the
  public contract.

## Durable compacted history

- **Conversation archives are async and host-injectable.** A
  `ChatArchiveRepository` stores immutable raw messages, rolling summaries, and
  the manifest that links them.
- **Archive commits have an explicit atomicity boundary.** `append` must make
  archive content and its next manifest visible together, preventing a durable
  manifest from referencing missing content.
- **Compaction restores through fresh service instances.** Heddle loads the
  repository-owned manifest and rolling summary during preflight and persists
  the session only after archive storage succeeds.
- **Local-first behavior remains the default.** Without a custom repository,
  Heddle uses its strict atomic file adapter under `stateRoot`.

## PostgreSQL reference

- A runnable PostgreSQL 17 + Drizzle example includes checked-in migrations,
  tenant-scoped session and archive repositories, revision compare-and-swap,
  stable cursor ordering, and transactional archive appends.
- The verification runs the public session conformance suite plus concurrent
  archive, corruption, tenant-isolation, and fresh-pool/engine recovery checks.
- Drizzle and `pg` remain example-only development dependencies. They are not
  Heddle runtime dependencies, exported adapters, or included compiled example
  code in the npm package.

## Upgrade notes

- Install `@roackb2/heddle@5.1.0` to use the public persistence codecs,
  conformance runner, or `archiveRepository` engine option.
- Existing `ChatSessionRepository` implementations remain compatible. Hosts
  can adopt the shared authoring helpers and conformance suite incrementally.
- Hosted services that need compacted histories to survive replica changes
  should bind `sessionRepository` and `archiveRepository` to the same trusted
  tenant scope. The default archive adapter still requires a shared durable
  `stateRoot`.
- `@roackb2/heddle-remote@5.1.0` is released in lockstep. Its hosted run
  protocol is unchanged.
