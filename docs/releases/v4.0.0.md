# Heddle v4.0.0

Heddle v4.0.0 turns the programmatic surface into a real SDK for building an
agentic experience into your own product. A third-party developer can now stand
up a conversational agent in a few lines, add their own tools and MCP servers,
shape streaming output, and â€” new in this release â€” back Heddle with their own
storage instead of a local state root. The public API is reorganized as a
progressive-disclosure ladder with a curated default entry and an `/advanced`
subpath for lower-level building blocks.

This is a major release because the public import surface changed (see Upgrade
Notes).

## Programmatic SDK

- **Host extensions.** `defineHostExtension(...)` composes your own tools,
  system context, and artifact support into a conversation engine. Host tools
  and toolkits can be injected directly.
- **MCP host extensions.** `prepareMcpHostExtension(...)` prepares an MCP
  server's catalog and exposes its tools as curated Heddle tools without
  copying schemas, with optional prefixing, overrides, hiding the raw `mcp_*`
  surface, and automatic capture of large results as artifacts.
- **Quickstart conversation runner.** `runQuickstartConversationCli(...)` owns
  the common host loop: persisted sessions, session resume, one-shot
  (`oncePrompt`) and scriptable multi-turn (`prompts`) runs, default text
  output, local commands, credential preflight, and lifecycle hooks.
- **Turn results.** Conversation turns return trace-file, artifact, and
  tool-result summaries, so hosts don't parse raw trace files.
- **Curated public surface.** The default `@roackb2/heddle` import is now the
  curated rung 1â€“5 SDK (start a conversation â†’ add capabilities â†’ shape output â†’
  lifecycle â†’ storage). Lower-level building blocks and specialized runtimes
  moved behind `@roackb2/heddle/advanced`.
- **Engine artifact reader.** `engine.artifacts` reads a session's artifacts
  through the engine's resolved root â€” no on-disk path reconstruction.

## Injectable storage (new)

- **`ArtifactRepository` port.** Back artifact catalog + content with your own
  storage; pass `artifactRepository` to `createConversationEngine(...)`. Used by
  the engine reader, turn-result listings, artifact tools, and MCP result-
  artifact capture.
- **`ChatSessionRepository` port.** Back the session catalog + bodies with your
  own storage; pass `sessionRepository`. Used across session lifecycle, turn
  preflight/persistence, leases, and background memory maintenance.
- Both default to the existing file-backed implementations, so local usage is
  unchanged. This is the seam a hosted service uses to embed Heddle without a
  local `.heddle/` directory.

## Fixes

- Tool policy envelope: accept empty `targetRoots` for non-mutating calls, and
  require at least one root for mutating calls (aligning the validator with the
  advertised schema and closing an autopilot gap).
- MCP result artifacts: compact duplicate serialized result mirrors into a
  single artifact reference; prefer hinted artifact paths.
- Browser automation: click/type actions.
- OAuth workspace auth fix.

## Upgrade Notes

Breaking changes to the public API (pre-stable SDK surface):

- **Subpath exports.** The default `@roackb2/heddle` entry now exports only the
  curated SDK. Lower-level symbols (LLM adapters and provider services,
  individual ready-made tools, trace/observability internals, memory services,
  agent loop, heartbeat, integrations, utilities) moved to
  `@roackb2/heddle/advanced`. Update imports of those symbols to the `/advanced`
  subpath.
- **Removed compatibility aliases.** `runConversationCli`,
  `resolveConversationCliDefaults`, `ConversationCliRunnerService`, and the
  `ConversationCli*` types were removed. Use the `Quickstart*` names
  (`runQuickstartConversationCli`, etc.).
- **Renamed repository methods.** `FileArtifactRepository.read()/write()` are now
  `readCatalog()/writeCatalog()` (part of the widened `ArtifactRepository`
  port).

Storage injection is opt-in; hosts that do nothing keep the file-backed
defaults. The storage ports are synchronous in this release; async support is
planned for database/remote-backed hosts.
