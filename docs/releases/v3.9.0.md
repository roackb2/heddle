# Heddle v3.9.0

Heddle v3.9.0 adds first-class local model support through Ollama. Users can
now run Heddle with installed local Ollama chat models, select those models from
terminal and browser model pickers, and verify local OpenAI-compatible behavior
with an opt-in smoke test.

## What's Changed

- Added an Ollama runtime provider that routes `ollama/<model>` selections
  through Ollama's local OpenAI-compatible endpoint.
- Added provider-owned Ollama model discovery so terminal `/model set` and the
  browser model selector can show installed local chat models when Ollama is
  running.
- Added shared model option resolution so CLI and web surfaces use the same
  server-backed model list instead of maintaining separate selector behavior.
- Added local provider credential handling for Ollama endpoint configuration,
  including `OLLAMA_OPENAI_BASE_URL`, `OLLAMA_BASE_URL`, and `OLLAMA_MODEL`.
- Added an opt-in `yarn smoke:ollama` script for checking local Ollama model
  listing, chat completion, and tool-call compatibility.
- Added documentation for local model setup in the README, zh-TW README, CLI
  reference, chat guide, development guide, and provider/model reference.

## Workflow Notes

Start Ollama locally, then choose an installed chat-capable model with the
`ollama/` prefix:

```bash
ollama list
heddle --model ollama/llama3.2:latest ask "Summarize this repository"
heddle chat --model ollama/llama3.2:latest
```

When Ollama is running, `/model set <query>` and the browser model selector can
discover installed local chat models from the local Ollama API.

Local model quality varies. Smaller or older local models may be slower, may be
weak at tool calling, may ignore tool results, or may produce confident but
wrong repository answers. Keep approval prompts enabled for risky work, review
traces and diffs carefully, and use stronger local or hosted models when
tool-calling reliability matters.
