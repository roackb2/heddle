# Heddle v0.0.30

Release date: 2026-04-26

## Summary

Heddle v0.0.30 improves the control-plane workflow for multi-workspace operation and review depth.

This release makes workspace context more explicit, routes control-plane surfaces into clearer screens, and upgrades review fidelity with structured diffs plus current-workspace patch visibility.

## Highlights

- Added routed workspaces view and stronger workspace switching flow in the control plane
- Bound agent tool execution to the selected workspace to reduce cross-workspace mistakes
- Expanded review surfaces with structured diff rendering and workspace-current diff visibility
- Improved control-plane review usability with refined tabs, workflows, and bounded refresh behavior
- Preserved live prompt input during session refresh so operator drafts are not lost
- Updated docs/screenshots and release auth guidance for smoother operator release execution

## What’s New

### Workspace-aware control plane

- Added routed workspaces screen and tab content routing improvements
- Improved workspace structure and workspace switching behavior in the control plane
- Bound agent tools to the selected workspace context

### Better review quality and navigation

- Added workspace diff review API and shared diff review domain
- Added structured diff rendering in review surfaces
- Added current workspace diff visibility during turn/session review
- Refined review tabs and overall review workflow interactions
- Added bounded refresh behavior for workspace review flows

### UX continuity and docs

- Preserved live prompt drafts through session refresh
- Refreshed screenshots and workspace/review documentation
- Added explicit GitHub release auth flow guidance for operators

## Upgrade Notes

- Package version is now `0.0.30`
- No breaking CLI flag or config migration is introduced in this release
- Operators should continue using the documented release sequence and verify `gh auth status` before GitHub release actions

## Known Limits

- The richer review surface improves patch visibility but is still focused on current trace-backed review context, not a full repository browser
- Workspace selection and review polish reduce operator friction, but broader multi-repo ownership semantics remain an ongoing runtime direction
