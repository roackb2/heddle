# Heddle v5.0.0

Heddle v5 establishes an asynchronous, record-oriented persistence boundary
for durable conversations. Local products retain a zero-configuration JSON
store, while hosted products can inject a database adapter without loading and
rewriting an entire session collection for each mutation.

This is a breaking SDK release. Conversation-session lifecycle methods are now
asynchronous, and custom `ChatSessionRepository` implementations must adopt the
new versioned record contract.

## Durable session storage

- **Async record operations replace collection rewrites.**
  `ChatSessionRepository` now exposes asynchronous `list`, `read`, `create`,
  `update`, and `delete` operations suitable for filesystem and database
  adapters.
- **Optimistic revisions prevent lost updates.** Stored sessions carry a
  monotonic revision. Updates and deletes require the expected revision, with
  typed already-exists and revision-conflict failures.
- **Catalog pagination is deterministic.** Adapters use an opaque cursor and a
  stable order across pinned state, update time, and session ID so tied rows do
  not disappear between pages.
- **The default JSON adapter is process-safe.** Immutable revision files,
  atomic catalog replacement, and inter-process locking preserve local-first
  durability without requiring a database.
- **Hosted storage remains product-scoped.** Heddle owns conversation records
  and lifecycle semantics; the host constructs each repository inside its
  authenticated tenant or workspace boundary.

## SDK-first guidance

- The root documentation now starts from embedding Heddle as a TypeScript
  runtime and SDK, with explicit ownership boundaries between Heddle and the
  host product.
- A new durable-session guide covers zero-configuration JSON storage, the
  custom repository contract, a practical PostgreSQL shape, compare-and-swap
  updates, and adapter verification requirements.
- Public storage types and errors are exported from the main package so hosts
  can implement and test production adapters directly.

## Upgrade notes

- Install `@roackb2/heddle@5.0.0` and await conversation-session lifecycle
  calls such as create, lookup, settings updates, archive, and delete.
- Replace custom whole-collection session stores with the new
  `ChatSessionRepository` methods and enforce expected-revision updates in the
  storage system itself.
- Products using Heddle's default file-backed session storage do not need to
  supply an adapter; keep `stateRoot` on a durable local filesystem.
- `@roackb2/heddle-remote@5.0.0` is released in lockstep. Its hosted run
  protocol is unchanged in this release.
