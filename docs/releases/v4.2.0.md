# Heddle v4.2.0

Heddle v4.2.0 adds a result-artifact mode for stateless MCP workflows and
tightens OpenAI reasoning request compatibility across model families.

This release is additive and backward compatible.

## Mirror MCP result artifacts without replacing tool output

- **Manual MCP result-artifact rules can now use `mode: 'mirror'`.** Heddle
  persists the selected result as an artifact while leaving the original value
  inline for the model and subsequent tool calls.
- **Why it matters.** Stateless MCP tools often take the current document as
  input and return the updated document. Replacement compaction breaks that
  edit chain; mirror mode preserves it while still giving the host a durable
  artifact, including support for `setCurrent: true` and
  `engine.artifacts.current(...)`.
- **Mirror rules compose safely with automatic capture.** A mirrored path and
  its descendants are excluded from the automatic replacement pass, so opting
  into auto capture cannot unexpectedly compact a value declared for mirroring.

## Safer OpenAI reasoning requests

- **Reasoning parameters now follow explicit model capabilities.** API-key
  models that do not support reasoning parameters no longer receive a
  `reasoning` request block that they would reject.
- **Reasoning summaries remain available on supported models.** Summary support
  is treated independently from configurable reasoning effort, preserving
  diagnostics for established reasoning models even when Heddle does not expose
  an effort setting for that model.

## Upgrade notes

- No migration is required.
- Existing MCP result-artifact rules keep replacement behavior by default.
  Hosts must opt into mirroring with `mode: 'mirror'` on a manual rule.
- No public APIs were removed or renamed.
