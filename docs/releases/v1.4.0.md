# Heddle v1.4.0

Release date: 2026-05-16

Heddle v1.4.0 is a foundation release for the conversation engine. It
consolidates the service structure behind sessions, turns, and compaction so
future refactors have a concrete pattern to follow instead of copying older
host-specific branches.

The main theme is architectural convergence: core chat behavior now has clearer
owners, more explicit contracts, and stronger browser/TUI verification.

## Highlights

### Conversation-engine services now have a reference structure

The session, turn, and compaction domains now follow a more consistent shape:
class-based services, typed contracts, focused subdomains, class-based
repositories, and schema/codec-owned persisted JSON validation.

Why this matters:

- future contributors have an obvious pattern for new core services
- service behavior is easier to find because it is no longer spread across
  one-off helpers and wrapper files
- host surfaces can keep moving toward calling core services instead of
  reaching into repositories or storage details

### Session persistence is stricter and easier to reason about

Saved chat-session catalog and body files now go through a dedicated codec and
Zod schemas with human-readable field descriptions. The old flat legacy session
format is no longer part of the supported loading path.

Why this matters:

- disk contracts are validated in one repository boundary
- parse behavior is easier to test and maintain
- the current catalog-plus-session-files format is now the documented source of
  truth

### Compaction is now an engine domain

Conversation compaction moved out of the old history module into its own engine
domain. The retained path is the LLM summarizer-based compaction flow, with
prompt, context building, token estimation, transcript rendering, and split
policy grouped under the compaction boundary.

Why this matters:

- compaction is now treated as a first-class engine behavior
- legacy deterministic compaction paths are removed from the active flow
- future compaction changes have a clearer owner and test surface

### Turn execution internals are more modular

Turn runtime resolution, context building, preflight, persistence, memory
maintenance, host projection, trace writing, and failure-message formatting now
live in explicit turn subdomains.

Why this matters:

- persisted turn behavior is easier to inspect phase by phase
- host adaptation and engine persistence are less tangled
- provider failure guidance now belongs to the turn failure domain instead of a
  flat chat-level helper

### Browser verification is now correctly named and runs on PRs

The current Playwright control-plane suite is now classified as browser
integration coverage. It runs against a real browser, Vite client, and fixture
daemon, while using a mocked agent path so it can run reliably on every PR.

Why this matters:

- `src/__tests__/browser-integration/` now describes what the suite really is
- `src/__tests__/e2e/` is reserved for future true end-to-end tests
- PR checks now cover the browser control-plane flow, including session create,
  send, continue, workspace switching, review surface, and mobile navigation

## Other Changes

- Added `@/` source path aliases across the build, test, lint, and web tooling
  surfaces.
- Documented the engine service pattern in `src/core/chat/`, the engine README,
  chat layering strategy, and agent context.
- Updated TUI verification fixtures to match the current session and compaction
  shapes.
- Updated public docs for current session storage, browser integration tests,
  and programmatic conversation-engine guidance.
