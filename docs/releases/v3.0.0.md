# Heddle v3.0.0

Release date: 2026-06-02

Heddle v3.0.0 makes the API-backed TUI v2 the default terminal experience and
moves the terminal, browser, and mobile surfaces onto the shared local
control-plane path. This is a major release because `heddle` and `heddle chat`
now open the new TUI by default instead of the legacy terminal UI.

## Highlights

### TUI v2 is now the default

Running `heddle` or `heddle chat` now opens the control-plane-backed TUI v2.
The legacy terminal UI route has since been removed from the public CLI surface
as part of the v1 retirement path.

Why this matters:

- terminal chat now uses the same API-backed session model as the browser UI
- session state, approvals, run status, and streamed responses share one
  control-plane contract
- users get the newer TUI without needing a separate `chat-v2` command
- the supported terminal entrypoints now stay on the shared control-plane path

### One local control plane can serve terminal, web, and mobile

The runtime registry and server lifecycle now support a workspace-agnostic
control-plane server that clients attach to by server identity and workspace
ID. If a live server exists, new clients attach to it. If the TUI starts first,
it can start the embedded control-plane server and the browser or mobile UI can
connect to that same server.

Why this matters:

- standalone daemon and embedded TUI startup no longer compete as separate
  ownership models
- workspace identity is passed by clients instead of being owned by the server
- shutdown clears only the server record owned by that process
- future clients can use the same server lifecycle path instead of inventing
  another local backend

### Conversations stream across devices

Terminal, browser, and mobile sessions now demonstrate the same conversation
and event stream updating across interfaces. The README includes new short
GIFs showing cross-device streaming, request approval, and diff review so new
users can see the product shape immediately.

Why this matters:

- users can submit or observe a run from multiple devices
- agent responses, status updates, and review context stay synchronized
- browser and mobile review become a practical companion to terminal work

### Browser control-plane reliability is improved

The browser UI now avoids opening unnecessary long-lived session streams for
inactive views and uses bounded request timeouts for control-plane requests.
This fixes a case where one busy Heddle tab could starve another tab and delay
initial page loading until the older tab was closed.

Why this matters:

- a fresh browser tab should load session content promptly
- session lists refresh when opening directly to a session URL
- submitting from web and observing from TUI share the same run path more
  predictably
