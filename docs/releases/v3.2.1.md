# Heddle v3.2.1

Release date: 2026-06-08

Heddle v3.2.1 is a focused control-plane usability and reliability release. It
keeps long-running slash commands from failing at the normal request timeout,
surfaces command failures in the terminal UI, and makes context-window overloads
recover through automatic compaction.

## Highlights

### Long-running slash commands get a larger request budget

Slash commands such as conversation compaction now use a longer request timeout
than ordinary control-plane queries.

Why this matters:

- commands that summarize or compact large sessions are less likely to fail at
  the previous 30 second request boundary
- command timeout policy is owned in the shared control-plane API path instead
  of being duplicated per UI
- subscriptions remain long-lived event streams and are kept outside finite
  request timeout handling

### Command failures are visible in the TUI

When a slash command fails, the terminal UI now surfaces the failure instead of
leaving the operator to infer what happened from a stalled or finished run.

Why this matters:

- users can see that a command failed and why
- command results and prompt error state stay aligned
- timeout failures are easier to diagnose without digging through hidden traces

### Context-window overloads trigger compaction

Conversation turns that fail with a clear context-window or context-length
overload now trigger forced compaction.

Why this matters:

- oversized sessions can recover automatically when the failure is clearly about
  model context capacity
- compaction stays in the conversation-engine persistence path rather than being
  guessed by terminal UI code
- future runs are less likely to keep failing on the same oversized transcript

## Upgrade Notes

- No command migration is required from v3.2.0.
- `heddle` and `heddle chat` continue to open the API-backed TUI.
- Browser and terminal clients continue to use the shared control-plane path.
