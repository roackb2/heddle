# Heddle v1.2.0

Release date: 2026-05-13

Heddle v1.2.0 expands coding situation awareness and continues the chat/runtime
cleanup work that makes session behavior easier to reason about and maintain.

The main theme is better project orientation without letting host-layer logic
keep spreading outward.

## Highlights

### Project dashboard now detects common coding project kinds

`project_dashboard` can now report bounded project signals for supported coding
workspaces instead of treating every repo like a generic tree listing.

Current supported detectors:

- JavaScript / TypeScript
- Python
- Go

The dashboard groups what it finds by detected project kind and exposes:

- manifests
- lockfiles
- verification surfaces
- observed source, docs, script, and config directories

Why this matters:

- fresh sessions can identify what kind of repo they are in much faster
- mixed-language workspaces are easier to inspect without blind file hopping
- the awareness output stays explicit, bounded, and traceable

### Chat session boundaries are cleaner

This release continues the chat session refactor by moving TUI session flows
behind core session services and introducing a real file-backed session
repository boundary.

Why this matters:

- TUI host code no longer owns file storage mechanics directly
- session behavior is easier to verify and evolve without touching multiple
  layers
- custom session catalog paths are preserved correctly instead of collapsing
  back to defaults

### TUI chat layering is easier to follow

The CLI chat host is now organized more clearly around:

- presentation in `App.tsx`
- controller-oriented React hooks for host-specific orchestration
- core/domain ownership for shared semantics

Why this matters:

- contributors have a clearer path for where new logic should live
- future situation-awareness work has a cleaner host-side foundation
- regression debugging should require less cross-layer archaeology

## Other Changes

- Added stronger detector/service README guidance for extensible coding project
  awareness.
- Expanded the TUI verifier script to cover custom session catalog path
  behavior.
- Added regression tests for session storage path handling and richer awareness
  payload coverage.
- Clarified release docs around common `gh` auth and tagging failure modes.
