# Heddle v1.1.0

Release date: 2026-05-12

Heddle v1.1.0 adds the first useful slice of situation awareness and tightens
chat session behavior so fresh sessions orient faster and session preferences
behave more predictably.

The main theme is better startup orientation with less rediscovery.

## Highlights

### Project dashboard for early situation awareness

Heddle now exposes a single `project_dashboard` awareness tool for coding
workflows. Instead of making the agent choose between several orientation tools,
one call can provide the current working environment and a bounded workspace
tree.

Why this matters:

- fresh sessions have one obvious way to orient before deeper inspection
- the awareness surface is simpler for agents to use reliably
- output stays bounded and inspectable instead of turning into hidden prompt
  injection

### Better fresh-session bootstrap behavior

The chat bootstrap guidance is now stricter about when a brand-new session
should call the dashboard before substantial planning, review, or coding work.

Why this matters:

- startup behavior is more reliable for naive or loosely structured asks
- the agent spends less time re-deriving repo state that it could get from the
  dashboard
- the awareness flow stays explicit in traces and tool history

### Session model and reasoning behavior is more predictable

This release fixes a class of session bugs around model and reasoning
preferences. Switching between sessions no longer overwrites the target
session's stored settings, and new sessions inherit the active session's
explicit preferences more consistently.

The default reasoning effort for GPT-5.4 and GPT-5.5 family models is also now
`medium` instead of `high`.

Why this matters:

- session switching is less surprising
- new sessions preserve the operator's active working mode better
- default runs are less likely to stall in long, low-action thinking loops

## Other Changes

- Added stronger chat layering guidance so domain policy stays out of the TUI
  presentation layer.
- Added a reusable maintainer verification script for TUI session-switch
  behavior.
- Improved situation-awareness tests around dirty git state, omitted paths, and
  bounded workspace-tree output.
