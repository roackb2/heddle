# Heddle v1.13.0

Release date: 2026-05-26

Heddle v1.13.0 is a feature release for workspace correctness in the browser
control plane. It tightens the meaning of a workspace around the `.heddle`
state root, adds a minimal web-v2 workspace settings surface, and makes session
runs, uploads, traces, logs, memory, tasks, and diff views follow the workspace
the operator is viewing.

## Highlights

### Workspace state roots are now the control-plane source of truth

The server now resolves a request workspace for workspace-owned control-plane
operations and routes session reads, session mutations, uploads, traces, logs,
memory, task state, and diff data through that workspace descriptor.

Why this matters:

- switching workspace in web-v2 changes the backend state root used by visible
  operations
- sessions can keep running in different workspaces without sharing the wrong
  state files
- workspace-specific logs are reused from a cached logger for consistent
  request and scheduler logging

### Web-v2 gets workspace switching and settings

The web-v2 sidebar now includes a compact workspace switcher, and settings has a
dedicated Workspace page for inspecting and creating workspaces. The settings
sidebar now shares the same width as the main workbench sidebar so moving between
the workbench and settings does not resize the shell.

Why this matters:

- workspace switching is available without bringing back the noisier v1
  workspace dashboard
- workspace management lives in settings instead of the main coding surface
- the shell stays visually stable while operators move between work and settings

### In-flight browser sessions survive navigation

Transient messages submitted from web-v2 are preserved while a session run is in
progress, even when the operator switches sessions or workspaces before the
assistant response completes.

Why this matters:

- the browser no longer loses the just-submitted prompt during common
  multitasking
- long-running sessions are easier to inspect while other workspaces remain
  available
- visible conversation state better matches the backend run state

### Web-v2 can stop a running session

The web-v2 session composer now exposes a stop control for active runs, backed by
the control-plane session cancellation path.

Why this matters:

- operators can stop a stuck or unwanted browser run without leaving the v2
  workbench
- run control behavior is covered by focused session-cancel tests
- the browser surface is closer to the terminal control workflow
