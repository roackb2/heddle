# Heddle v2.5.0

Release date: 2026-06-02

Heddle v2.5.0 adds direct shell command support to the API-backed control
plane and cleans up the cli-v2 session store into smaller workflow owners. This
release keeps web-v2 and `chat-v2` on the same control-plane path for shell
commands while making the terminal session state easier to maintain.

## Highlights

### Direct shell commands work through the shared control plane

Users can run direct shell commands from web-v2 and `chat-v2` with a shared
control-plane execution path. Shell policy remains backend-owned, while each
interface renders its own confirmation and result UI.

Why this matters:

- web-v2 and `chat-v2` use the same API contract for direct shell behavior
- simple inspection commands can run without unnecessary friction
- risky or unclassified commands still surface explicit user confirmation
- shell command results are structured for interface-specific rendering

### Direct shell mode is clearer in both interfaces

When a prompt starts with `!`, web-v2 and `chat-v2` now make the direct shell
mode visible before the command is submitted. This reduces accidental shell
execution while preserving a fast command workflow.

Why this matters:

- users can see when they are typing a local shell command
- confirmation remains a front-end UX layer over backend policy
- command output is visible as part of the conversation flow

### cli-v2 session state has clearer workflow owners

The large cli-v2 control-plane session store has been split into focused state
and workflow modules for loading, prompt routing, slash commands, direct shell,
approvals, run polling, and live-event reduction. The public store remains the
Ink-facing facade, but private forwarding wrappers were removed so ownership is
explicit.

Why this matters:

- future cli-v2 changes have clearer extension points
- domain policy stays in core, control-plane, or client-shared owners
- the terminal UI keeps one render snapshot without making one class own every
  workflow detail
