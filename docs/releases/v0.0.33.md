# Heddle v0.0.33

Heddle v0.0.33 improves real operator follow-through, adds direct file cleanup/move tools, and makes OpenAI account-sign-in more useful by enabling `view_image` through the Codex OAuth transport.

## Highlights

- Added OpenAI OAuth / Codex support for `view_image`
- Added first-class `delete_file` and `move_file` tools for direct workspace cleanup and renames
- Removed the hardcoded `Changed / Verified / Remaining uncertainty` final-answer template enforcement
- Softened continuation and blocker guidance so Heddle is more likely to carry bounded work through instead of pausing for repetitive checkpoint narration
- Updated user-facing docs to reflect the new `view_image` and file-operation behavior
- Improved browser diff/review fidelity with the new Monaco diff review work and related control-plane updates already included in this release range
- Added mocked Playwright control-plane session flow coverage in the release scope since `v0.0.32`

## Why it matters

This release makes Heddle feel more like an actual coding/workspace operator and less like a tool wrapper that needs repeated nudges.

The main behavior improvements are practical:

- image inspection now works under OpenAI account sign-in in the same general Codex path many operators already expect
- routine file cleanup and rename work no longer needs shell mutation as the first option
- final answers after change work can stay natural instead of being forced into a rigid canned template
- blocker handling is still explicit, but `report_state` is now framed more like a genuine blocker tool than a progress ritual

## Operator notes

- OpenAI account sign-in now supports `view_image` through the Codex OAuth transport.
- OpenAI hosted `web_search` and drift embeddings still require Platform API-key mode.
- `delete_file` and `move_file` are approval-gated first-class tools for cleanup and renames.
- Heddle now prefers more natural completion behavior, but still expects grounded verification and repo review after real workspace changes.
- This release range also includes the Monaco diff viewer and additional control-plane review improvements, so browser-side change inspection is stronger than in `v0.0.32`.

## Upgrade notes

- No breaking CLI migration is required for this release.
- Existing API-key workflows continue to work unchanged.
- Operators using OpenAI OAuth for screenshot or UI review can now rely on `view_image` without switching to API-key mode for that capability.
- If you have local scripts or habits that assumed shell `mv`/`rm` for simple workspace cleanup, the direct tools are now available and should usually be preferred.

## Remaining limits

- OpenAI hosted `web_search` still requires Platform API-key mode.
- Drift embeddings remain API-key-only under the current OpenAI path.
- Memory retrieval/discovery smoothing is still an active follow-up area and is not fully solved in this release.
