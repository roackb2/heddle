# Heddle v0.0.28

Release date: 2026-04-21

## Summary

Heddle v0.0.28 turns the control plane into a more practical local workstation surface, adds a real source-based daemon development flow, and relaxes chat ownership so terminal chat, browser, and `ask` can coexist more naturally in one workspace.

## Highlights

- `heddle daemon` now has a clearer local development path with `yarn daemon:dev`, built asset serving, and more reliable shutdown/ownership cleanup
- `heddle ask` can attach to a live daemon owner instead of failing on workspace ownership conflicts
- Embedded terminal chat is no longer blocked just because a daemon is running in the same workspace
- Chat mutation now uses a lightweight per-session lease, so the conflict boundary is the session rather than the whole workspace
- The browser and mobile control plane now read more like a workstation shell and less like a metadata dashboard

## What's New

### Local workstation and daemon workflow

- Added workspace catalog foundations, workspace creation/switching, and explicit workspace identity in the control plane
- Added user-level daemon registry integration so Heddle can discover active workspace ownership
- Added `yarn daemon:dev` for running the real daemon path from source while serving built control-plane assets
- Fixed daemon shutdown so `Ctrl+C` properly releases ownership and closes the server instead of leaving stale listeners behind

### Daemon-aware clients

- `heddle ask` now attaches to a live daemon owner for both stateless asks and session-backed asks
- The browser control plane surfaces daemon-backed workspace state without requiring a separate mental model per device
- Desktop web and mobile web are now documented explicitly as daemon clients, not separate owners

### Session-scoped chat safety

- Embedded terminal chat can now run alongside a live daemon in the same workspace
- Chat mutation safety moved away from workspace-wide exclusion toward lightweight per-session leases
- Different sessions can coexist in one workspace across TUI, web, mobile, and `ask`
- The guarded case is now concurrent mutation of the same session

### Control plane refinement

- Mobile control plane received a tighter shell, compact composer, native-style session navigation, and improved overview layout
- Desktop/mobile host chrome was simplified after review so the shell focuses on actual work instead of runtime badges and control surfaces
- Added debug snapshot support for the browser control plane and terminal chat to help inspect real UI state during debugging

## Fixes

- Direct daemon origin serving now resolves asset paths correctly
- Daemon ownership no longer appears falsely live when the recorded PID has already exited
- Restarting a dev daemon after `Ctrl+C` no longer leaves the old server holding the port
- Control-plane mobile layout and overview spacing were tightened substantially

## Upgrade Notes

- No breaking CLI command removals in this release
- Package version is now `0.0.28`
- If you operate directly on saved chat session files, note that session records now include lightweight lease metadata
- The intended local host-model dev workflow is:
  - `yarn build`
  - `yarn daemon:dev`
  - optional `yarn client:dev` if you want Vite against the same daemon

## Known Limits

- Session lease protection is intentionally lightweight; it blocks concurrent mutation of the same session but does not provide richer presence or takeover UX
- Terminal chat still does not attach to daemon-owned sessions as a remote interactive client
- Browser/mobile host actions remain intentionally minimal until there is a real user-facing ownership-transfer need
