# Heddle v5.4.0

Heddle v5.4 gives product hosts clearer, live evidence of what an agent is
doing during longer turns. Provider reasoning summaries and assistant-authored
work commentary now travel as distinct activities, so each interface can
present useful progress without exposing hidden chain-of-thought or confusing
an in-progress message with the final answer.

## Reasoning-summary streaming

- **Reasoning summaries stream as their own activity.** Supported OpenAI
  reasoning models emit cumulative `reasoning.summary` text and an explicit
  completion marker independently from the final answer.
- **Codex account requests ask for detailed summaries.** Heddle includes the
  encrypted reasoning item required by the Codex transport and requests a
  detailed user-visible summary instead of relying on an optional automatic
  summary.
- **Presentation remains host-owned.** Core emits raw summary text without a
  `Thinking` prefix; CLI, web, and embedded products decide how to label,
  collapse, or omit it.

## Assistant work commentary

- **Assistant-authored progress is distinguishable from reasoning.** Codex
  messages with `phase: "commentary"` stream as `assistant.commentary`
  activities before or between tool calls.
- **Commentary remains separate from the terminal response.** Heddle correlates
  the output-item phase with its text stream by message ID and does not infer
  meaning from wording or event order.
- **Hosts receive stable structured fields.** Each commentary activity carries
  a message ID, cumulative text, step, and completion state through the normal
  ordered conversation-run stream.

## Interface and host behavior

- Heddle web and terminal clients can render reasoning summaries, assistant
  commentary, tool progress, and the final answer as separate user-facing
  concepts.
- Embedded hosts can surface the same activities without enabling the
  incomplete final-answer draft stream.
- Generic in-progress status remains the fallback when a provider emits no
  summary or commentary for a short turn.

## Upgrade notes

- Install `@roackb2/heddle@5.4.0` to receive `reasoning.summary` and
  `assistant.commentary` conversation activities.
- Hosts should render these activity types independently. Neither one is hidden
  chain-of-thought, and neither should be treated as the terminal answer.
- Existing API-key, request-scoped access-token, and stored-OAuth credential
  paths remain compatible.
- `@roackb2/heddle-remote@5.4.0` is released in lockstep. Its ordered run and
  reconnect protocol remains compatible.
