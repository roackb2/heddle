# Heddle v0.0.36

Heddle v0.0.36 is a release focused on keeping real coding sessions moving while improving confidence in the workflow around them.

This release does three practical things:
- it makes OpenAI account-sign-in mode work for hosted `web_search`
- it removes a couple of agent-runtime heuristics that previously escalated into hard blockers during iterative work
- it adds a GitHub pull-request quality-check workflow so future changes are verified more consistently before merge

## Highlights

### OpenAI OAuth web search now works in practice

Heddle can now use OpenAI hosted web search when the active OpenAI credential is ChatGPT/Codex account sign-in rather than only Platform API-key mode.

This includes:
- OAuth wiring for `web_search`
- Codex-compatible SSE parsing for final summary text and consulted sources
- a small shared helper refactor so the same OAuth/Codex stream-handling pattern is reused across `web_search` and `view_image`

Why this matters:
- account-sign-in mode is now more viable for normal operator use
- the hosted `web_search` tool behaves more like the rest of the OpenAI OAuth path instead of being a special failure case

### Repeated retries warn instead of blocking the run

The agent runtime no longer hard-stops on two common iterative-work patterns:
- repeated identical tool calls
- three consecutive tool errors

Instead, Heddle now warns the agent in-context and lets it continue, so it can try a different approach, narrower scope, or a more appropriate tool without being cut off by the runtime.

Why this matters:
- real debugging often requires retries
- operator-directed persistence is less likely to get turned into an artificial blocker
- warning-only guidance keeps the agent steerable without making it inert

### Pull requests now run automated quality checks

This release adds a GitHub Actions workflow for pull requests targeting `main`.

The workflow runs:
- eslint
- typecheck
- tests

These checks run as separate jobs, with cache-based installs per job.

Why this matters:
- merge-time verification is more consistent
- workflow regressions are more likely to be caught before they land on `main`

### Memory candidate source references are stricter

`record_knowledge` sourceRef validation is tighter now.

This release rejects more clearly unsafe provenance references such as:
- absolute paths
- upward traversal outside the workspace
- URI-style refs

Explicit non-path runtime references such as `trace-*`, `session-*`, and `command:*` remain allowed.

## Operator notes

- No breaking CLI migration is required for this release.
- The new PR workflow improves future release confidence, but it does not replace the normal release verification baseline.
- OpenAI OAuth web search now behaves like a supported path, but it still depends on the hosted provider behavior and current model/tool support.

## Verification baseline for the release candidate

The release convention expects the normal green checkpoint baseline:
- `yarn build`
- `yarn test`
- `npm pack --dry-run --cache /tmp/heddle-npm-cache`

## Scope

This note is based on the real git range from `v0.0.35` to `HEAD`, including:

- memory retrieval guidance update
