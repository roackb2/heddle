# Heddle v4.3.0

Heddle v4.3.0 makes reconnectable, long-lived conversational agents a
first-class SDK workflow and introduces a lightweight package for remote
browser clients.

This release is additive for previously published APIs. Server hosts can use
the new run lifecycle without adopting a transport, while remote clients can
install the protocol layer without pulling in Heddle's Node runtime, CLI,
model providers, server, or control plane.

## Host long-lived conversational runs

- **`ConversationRunService` owns active-run coordination.** A host can start a
  turn, return one stable run ID, subscribe separately, reconnect from an
  accepted sequence, cancel explicitly, and resolve approvals without building
  a second event bus around the conversation engine.
- **Ordered replay and terminal semantics are built in.** Runs publish canonical
  activity, result, cancellation, and error events with bounded replay and one
  monotonic sequence.
- **Hosted assumptions are explicit.** Import the process-local coordinator
  from `@roackb2/heddle/hosted`; authentication, routing, transport, durable
  delivery, and product result projection remain host-owned.

## Lightweight remote client package

- **`@roackb2/heddle-remote` is independently installable.** It contains the
  transport-neutral run codec and consumer state machine with only Zod and
  Standard Schema as dependencies.
- **The protocol codec validates untrusted envelopes and host payloads.** Hosts
  provide synchronous Standard Schema validators, so Zod 3.24+, Zod 4,
  Valibot, ArkType, and other compatible validators can define the authorized
  public activity/result shape.
- **The consumer owns reconnect correctness.** It advances only the greatest
  accepted sequence, suppresses replay duplicates, rejects gaps and
  post-terminal events, recognizes terminal outcomes, and calculates bounded
  exponential retry delays without choosing HTTP, SSE, tRPC, WebSocket, React,
  or timers.
- **Public payload projection stays explicit.** The codec enforces JSON safety
  after validation; examples now allowlist public activity fields instead of
  passing internal trace or tool state through a permissive schema.

## One lifecycle inside and outside Heddle

- **CLI-v2 and web-v2 now consume the same run lifecycle exposed to SDK
  users.** Run identity, ordered replay, cancellation, approval resolution,
  reconnect cursors, and terminal handling no longer diverge between Heddle's
  own interfaces and embedded products.
- **The previous control-plane-only run coordinator and duplicate client stream
  state machine are removed.** Shared behavior lives in the core run and remote
  domains, while each interface keeps only presentation and transport work.

## Stronger embedded-host ergonomics

- **Engine-level `toolProfile` controls model-visible defaults.** Embedded hosts
  can remove Heddle-managed capabilities such as memory tools while retaining
  host extensions; a selected custom agent still has turn-level precedence.
- **`engine.sessions.readExisting(id)` performs a persisted-only lookup.** Hosts
  can resume deterministic product conversations without materializing a
  fallback session in an empty repository.
- **`HeddleEventType` is available from the curated entrypoint.** Host event
  adapters no longer need to duplicate activity-name strings.
- **A complete numbered hosted-agent example is included.** It progresses from
  a transport-neutral application service to an authenticated Express/SSE API
  and a framework-neutral browser client, with responsibility and deployment
  assumptions documented at each layer.

## Upgrade notes

- Existing `@roackb2/heddle` imports remain valid.
- Server hosts that need addressable active runs should import
  `ConversationRunService` from `@roackb2/heddle/hosted`.
- Remote browser clients should install and import from
  `@roackb2/heddle-remote`; they no longer need the root Heddle package.
- The temporary root `/remote` subpath was developed and corrected before an
  npm release, so no previously published import path is removed.
- `@roackb2/heddle` and `@roackb2/heddle-remote` initially release in lockstep
  at `4.3.0`.
