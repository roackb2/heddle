# Heddle v0.0.32

Heddle v0.0.32 adds experimental OpenAI account sign-in, makes auth choice more visible in terminal and browser session flows, and tightens the runtime so OAuth-backed Codex sessions behave more like normal day-to-day coding runs.

## Highlights

- Added experimental OpenAI account sign-in with `heddle auth login openai`
- Added chat slash commands for auth status, login, and logout
- Added explicit auth source indicators in terminal chat and the browser control plane session composer
- Added `--prefer-api-key` so you can keep OAuth stored but force Platform API-key mode for testing
- Fixed OAuth Codex request handling so streamed tool calls, system instructions, and allowed model routing work correctly
- Disabled drift by default for new sessions and blocked OAuth sessions from silently using Platform API credits for drift embeddings

## Why it matters

This release makes Heddle easier to operate when you want to use OpenAI through your own ChatGPT/Codex account instead of only through Platform API keys. It also makes the active credential path more inspectable, which matters when you are trying to avoid accidental API-key usage during normal coding work.

The OAuth transport path now behaves much closer to the existing API-key path: tool calls stream correctly, Codex requests use the expected payload shape, and unsupported account-sign-in cases fail more explicitly instead of looking like random runtime breakage.

## Operator notes

- OpenAI account sign-in is experimental, user-selected, and not official OpenAI support.
- Heddle still supports normal OpenAI Platform API-key mode with `OPENAI_API_KEY`.
- If both OAuth and an OpenAI API key are present, Heddle prefers OAuth by default. Use `--prefer-api-key` when you want a run or daemon session to use Platform API-key mode instead.
- OpenAI hosted `web_search`, `view_image`, and drift embeddings still require Platform API-key mode.
- Anthropic remains API-key only. Heddle does not support Anthropic consumer subscription OAuth.

## Upgrade notes

- No breaking CLI migration is required for this release.
- Existing users who already export OpenAI API keys can keep their current workflow unchanged.
- Users who test both auth modes should pay attention to the footer auth indicator in chat or the session composer auth indicator in the control plane.
